Winter Messages

This project is a real-time chat application built using React and Firebase Firestore. It showcases my ability to create interactive and dynamic web applications with a focus on real-time data synchronization. The application allows users to engage in conversations, with each message update being instantly reflected across all connected clients. Utilizing Firebase's powerful backend services, the app handles user authentication and real-time database updates seamlessly. The project demonstrates my proficiency in using modern web development tools and frameworks to create a responsive and user-friendly chat interface.

Art Higher or Lower

Created a higher or lower game based on the price of popular paintings using the PERN stack (PostgreSQL, Express.js, React.js, Node.js). When a user starts a game they are presented with 2 paintings and must click the higher or lower button based on the value of the painting. If they think the value of the painting on the left has a higher value than the painting on the right, they must click the higher button. This is still a work in progress and I hope to implement an account system and leaderboard system to keep track of the person with the longest streak.


406 Trades

Role: Back-end Developer/Database Admin

406 Trades was a stock simulation application created by myself and 5 others. The website was written in Python using Flask. We used PyMongo to communicate with our MongoDB database to store users' account information. We used Alpaca API to get real-time stock information.

Users would create accounts and start off with some capital. They could then invest in different stocks and view their portfolio over time.


Multiverse Adventure

Role: Owner, Full-Stack Developer

Multiverse Adventure is a Roblox simulator/tycoon hybrid where players must advance through different worlds. All Roblox games are coded in LUA. I worked on creating 3D models for the game as well as scripting many core functional elements. We utilize client-server architecture to keep most of the work on Roblox servers to reduce the load on the clients. This allows users with under-powered computers to be able to enjoy our game.

Obstacle course simulator

Created an obstacle course where users would jump and progress through different levels. Users would reach a checkpoint at the end of each level and their progress was saved into a datastore. Players will also able to skip levels by paying a fee. Each time a player finished the entire obstacle course, they can restart the game but with increased speed and jump height.


Infinity Obby

Created an infinitely generating game where players must complete different stages to unlock higher difficulties. Stages of the game are procedurally generated and go on infinitely. I worked on creating the script that handels all of the player interactions with the stages. I 3D modeled all of the stages and the star islands which contain stars that allow players to skip stages. I designed a user-friendly UI that players can interact with to see their progress or change their difficulty.
